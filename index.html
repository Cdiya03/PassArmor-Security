    CRYPTOGRAPHY
    What is Shannon Entropy and how does it apply to passwords?
    ▼
    EXPLANATION VECTOR // CRYPTOGRAPHY

    Shannon Entropy measures the mathematical unpredictability of a password string in bits. The formula is H = L × log₂(R) where L is length and R is character pool size. Each bit added doubles the cryptographic difficulty. High-security criteria standard is 80+ bits of entropy.

    Academic Tip: During viva, reference that PassArmor mitigates human pattern faults by checking for keyboard walks and sequential chars, conforming to NIST rules rather than old character complexity rules.
    STANDARDS
    Why does NIST SP 800-63B recommend length over complexity rules?
    ▼
    EXPLANATION VECTOR // STANDARDS

    NIST SP 800-63B (2017 revision) removed mandatory complexity rules after research showed they produced predictable patterns. Length increases entropy more efficiently than complexity because each additional character multiplies search space by the pool size, while users tend to substitute characters predictably (e.g. @ for a, 3 for e).

    Academic Tip: Cite that NIST now recommends minimum 8 chars (15 for privileged accounts) with no mandatory complexity, but allows user-selected passphrases. PassArmor checks both.
    THREAT VECTORS
    What is a dictionary attack and how does PassArmor analyze it?
    ▼
    EXPLANATION VECTOR // THREAT VECTORS

    A dictionary attack uses pre-compiled wordlists of common passwords rather than pure brute force. PassArmor's analyzer detects common patterns: sequential characters (abc, 123), keyboard walks (qwerty, asdf), repeated characters, and single dictionary words. These heuristics flag high-risk inputs that would fall to dictionary attacks in milliseconds.

    Academic Tip: Distinguish from brute force — dictionary attacks prioritize likely candidates. PassArmor's crack matrix models worst-case brute force, so real-world time against common passwords is much lower.
    IMPLEMENTATION
    How is security and zero credential leak guaranteed in PassArmor?
    ▼
    EXPLANATION VECTOR // IMPLEMENTATION

    PassArmor processes all password analysis entirely within the user's browser using JavaScript. No network requests are made that include user credentials. The password generator uses window.crypto.getRandomValues() — the browser's cryptographically secure pseudorandom number generator. No server logs, no telemetry, no third-party calls.

    Academic Tip: This is the privacy-by-design principle — security built into the architecture rather than applied afterwards. Can be verified by opening browser DevTools Network tab and observing zero outbound requests during analysis.
    PASS GEN
    Why should passphrases be preferred over random character blocks?
    ▼
    EXPLANATION VECTOR // PASS GEN

    Passphrases (e.g. "galaxy-phantom-orbit-titan") combine high memorability with strong entropy. A 4-word passphrase from a 7776-word Diceware list has ~51 bits of entropy; a 6-word passphrase achieves ~77 bits. Random character strings have higher bits-per-character but are difficult to memorize, leading users to write them down — creating physical security risks.

    Academic Tip: Reference XKCD 936 "correct horse battery staple" for viva. The passphrase mode in PassArmor uses a curated wordlist with crypto.getRandomValues for unbiased selection.
